# prompt-hub

Единый дашборд по трём SEO-проектам (shustrik-maps.com, itc.by, fit.shustrik-maps.com):
таблица страниц/карточек со статусом, кнопка "Промпт" генерирует готовый текст для LLM
на основе данных строки (свой стиль промпта под каждый сайт), кнопка статуса отмечает
прогресс.

## Что внутри
- `app/server.py` — Flask-приложение (роуты, БД, генерация промптов)
- `app/db.py` — схема SQLite
- `app/import.py` — импорт из трёх xlsx в БД (сохраняет прогресс Applied? при повторном импорте)
- `app/prompts.py` — 3 шаблона промптов: `commerce_en` (shustrik-maps, английский),
  `b2b_ru` (itc.by), `personal_ru` (fit.shustrik-maps)
- `app/templates/index.html` — дашборд
- `import_data/` — исходные xlsx (кладите сюда актуальные версии с каждого проекта)
- `data/` — SQLite база (создаётся автоматически, переживает пересоздание контейнера)

## Деплой на сервере (под ваш процесс)

```bash
cd /root/prompt-hub
git pull
docker compose up -d --force-recreate
```

**Важно про кэш:** код (`app/`) и шаблоны смонтированы как volume, а не запечены в
образ — поэтому `--force-recreate` без `--build` всё равно подхватывает свежий код.
Пересборка образа (`docker compose build`) нужна только если менялся
`app/requirements.txt` (новая зависимость).

Первый запуск:
```bash
cd /root/prompt-hub
docker compose up -d --build
```
При первом старте контейнер автоматически создаст базу и импортирует все три xlsx
из `import_data/`.

Приложение слушает **порт 8030** (проверил по вашей таблице — свободен).

## Как обновлять xlsx-данные

1. Обновили xlsx одного из проектов (после новой SEO-сессии) → положите свежий файл
   в `import_data/`, заменив старый (имя файла должно совпадать, см. `app/import.py`,
   словарь `SITES`)
2. `git add . && git commit -m "update data" && git push`
3. На сервере: `git pull`, затем нажмите кнопку **"Обновить данные из xlsx"** в самом
   дашборде (не нужно перезапускать контейнер) — либо `docker compose restart prompt-hub`
4. Прогресс (Applied?/Notes) сохраняется по ключу URL/название — повторный импорт его
   не затирает

## Кнопка "Экспорт {sайт}.xlsx"

Выгружает текущее состояние таблицы (включая отметки Applied?) обратно в xlsx — чтобы
можно было закоммитить актуальный прогресс в соответствующий репозиторий проекта
(`shustrik-maps-seo`, `itc-by-seo`, `fit-shustrik-maps-seo`) вручную, как вы делали раньше.

## Безопасность

Приложение не имеет доступа к GitHub API и не хранит никаких токенов — весь git
pull/push вы делаете сами на сервере через уже настроенный у вас доступ. Открыто без
авторизации (просто по адресу `http://147.45.42.169:8030`) — если нужен пароль на
вход, добавьте basic-auth на уровне nginx/Caddy перед этим сервисом, либо скажите,
добавлю Flask-Login на следующем шаге.

## Локальный запуск без Docker (для теста)

```bash
cd app
pip install -r requirements.txt
python import.py
python server.py
```
