🧩 Промпт-шаблоны

← назад к таблице
Для каждой пары «сайт + тип страницы» можно сохранить кастомный Jinja2-шаблон промпта — он подставит данные строки сами и будет использован вместо общего движка, если у строки в таблице выбран этот тип страницы. Если для типа кастомного шаблона нет — работает обычная генерация как раньше.

Доступные переменные в шаблоне:
{{ '{{ row.name }}' }}, {{ '{{ row.primary_keyword }}' }}, {{ '{{ row.secondary_keywords }}' }}, {{ '{{ row.seo_title }}' }}, {{ '{{ row.h1 }}' }}, {{ '{{ row.meta_description }}' }}, {{ '{{ row.lsi_keywords }}' }}, {{ '{{ row.faq_questions }}' }}, {{ '{{ row.notes }}' }}, {{ '{{ row.url }}' }} — любое поле строки напрямую;
{{ '{{ breadcrumb }}' }} — хлебные крошки раздела структуры (если страница привязана);
{{ '{{ related_block }}' }} — готовый текстовый список реальных связанных страниц; {{ '{{ has_related }}' }} — есть ли они (для {{ '{% if has_related %}' }});
{{ '{{ input_lines }}' }} — тот же список строк входных данных, что и в обычном движке (можно вывести циклом {{ '{% for line in input_lines %}' }});
{{ '{{ cfg.persona }}' }}, {{ '{{ cfg.banned_words }}' }}, {{ '{{ cfg.tone_bullets }}' }} и другие поля конфига сайта (persona/audience/ eeat_frame/local_seo/no_invent/...) — та же структура, что в prompt_config/<site>.yaml;
{{ '{{ limits.seo_title_chars }}' }} и другие лимиты из _shared_defaults.yaml.
{% for s in sites %}{% endfor %} {% for code, label in page_types %} {% for s in sites %} {% set key = s.id ~ ':' ~ code %} {% set t = existing_map.get(key) %} {% endfor %} {% endfor %}
Тип страницы{{ s.name }}
{{ label }}
{{ code }}		 {% if t %}
обновлён {{ t.updated_at }}
{% endif %}

Jinja2-шаблон. Заголовки разделов начинайте с # — это Markdown.
