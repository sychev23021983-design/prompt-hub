# Changelog — SEO Content Engine (prompt_config / prompt_templates)

Версионируется контентная часть системы генерации промптов: стиль, лимиты,
чек-лист, формат вывода. Логика данных (Flask/SQLite) версионируется вместе
с остальным кодом репозитория и сюда не включается.

## v1.0 — 2026-07-11
Базовая версия, перенесена из захардкоженного `prompts.py` без изменения
поведения (проверено построчным сравнением вывода для всех трёх сайтов).

- ✓ 3 конфига сайтов: `commerce_en`, `b2b_ru`, `personal_ru`
- ✓ Общий шаблон `master_prompt.md.j2` + общий чек-лист `_shared/quality_check.md.j2`
- ✓ Общие числовые лимиты в `_shared_defaults.yaml` с возможностью override по сайту
- ✓ EEAT-тон, анти-AI-штампы, запрет на выдумывание фактов
- ✓ Локальное SEO (Минск/Беларусь)
- ✓ FAQ, Schema.org, Open Graph, 2-3 изображения с ALT/Title/caption
- ✓ Самопроверка по Yoast перед выдачей ответа

## v1.1 — 2026-07-11
По итогам разбора переписки с обсуждением доработок промпта (структурированные
входные данные, числовые лимиты, стоп-лист, перелинковка, вариативность). Внедрено
на все 3 сайта (движок общий, не только для WordPress+Yoast-сайта из обсуждения).

**Обязательные из обсуждения:**
- ✓ Явная разметка "нет данных" в блоке входных данных вместо тихого пропуска
  поля (раньше пустое поле в БД могло даже вывести буквальное "None" в промпт)
- ✓ Числовые лимиты для технических полей: SEO Title 50-60, H1 до 70, Meta
  140-160 (было и раньше), ALT до 125, Title изображения 40-70
- ✓ Запрет на выдумывание URL для перелинковки — теперь явный, плюс раздел
  "Рекомендуемые внутренние ссылки" в выводе не создаётся вообще, если
  реальных связанных страниц нет (has_related)
- ✓ Отдельная таблица чек-листа по стоп-словам/AI-штампам (генерируется
  динамически из banned_words/ai_openers конфига сайта) — в дополнение к
  прозе "никогда не используй"

**Желательные из обсуждения:**
- ✓ Гибкое количество FAQ (обычно 4-6, не жёсткий диапазон) вместо "6-10"
- ✓ Вариативность структуры страниц: конкретный список "подходов к подаче"
  (page_angles) на выбор модели вместо метафоры "представь 30 страниц"
- ✓ Типизация темы страницы перед написанием (page_types) — пока только для
  personal_ru, где это давало наибольший эффект (TRX ≠ здоровая спина)
- ✓ Явный запрет "не старайся использовать все ключевые слова любой ценой"
- ✓ Объём основного текста — качественный ориентир ("тема определяет объём",
  обычно 400-900 слов) вместо жёсткого "150-250 слов"

**Не изменилось (сознательно):**
- Числовые лимиты остаются числами именно там, где это технически обосновано
  (поля, которые обрезаются в выдаче) — гибкость только для объёма текста/FAQ,
  где жёсткие числа скорее вредят качеству

**Убрано:**
- ✗ Schema/JSON-LD секция — решили не дублировать то, что уже генерирует Yoast

## Как редактировать
- Изменить тон/персону/банворды одного сайта → правьте `prompt_config/<style>.yaml`
- Изменить общий чек-лист или формат вывода для всех сайтов → правьте файлы в
  `prompt_templates/`
- Изменить лимиты (длины полей, кол-во FAQ/картинок) → `prompt_config/_shared_defaults.yaml`,
  либо `overrides:` в конфиге конкретного сайта, если нужно только для него
- Изменения применяются на сервере через `git pull` — пересборка контейнера не нужна
  (код смонтирован как volume)
